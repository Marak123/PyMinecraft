# Phase 3: Texture System Upgrade

**Goal:** Replace the 16×16 procedural-only texture system with a high-resolution, multi-layer system supporting PBR maps, connected textures, and optional external texture pack loading.

**Depends on:** Phase 2 (PBR shaders need normal + MRAO maps to sample).

---

## Current State Analysis

**File:** `engine/graphics/atlas.py` (~620 lines)
- Generates ~45 tile layers as a `ModernGL TextureArray` at 16×16 resolution.
- Each tile is painted pixel-by-pixel in Python (noise-based patterns for stone, gradient for grass, etc.).
- Tile names are mapped to layer indices via a dict.
- Mipmaps are generated by ModernGL.
- UI icons (hearts, bubbles, hotbar) are also stored in the same atlas.

**File:** `configs/blocks.json` (36 blocks)
- Each block specifies tile names: `"textures": {"all": "stone"}` or `{"top": "grass_top", "side": "grass_side", "bottom": "dirt"}`.
- No normal/roughness/metallic texture references.

---

## Implementation Steps

### 3.1 Increase Procedural Texture Resolution [MODIFY atlas.py]

Upgrade all procedural tile generators from 16×16 to **128×128**:

- All existing painting functions (stone, grass, dirt, ores, wood, leaves, etc.) must be rewritten to produce 128×128 detail. The current noise patterns will look blurry if simply upscaled — they need higher-frequency noise octaves.
- **Performance consideration:** Generating 45+ tiles at 128×128 in pure Python will be slower (~2–5 seconds at startup). If this is too slow, use NumPy vectorized operations (already partially used) or Numba JIT for the noise fills.
- Separate UI icon generation from block textures — UI icons can remain 16×16 (they're rendered at screen-pixel scale anyway). Create a second small `TextureArray` or `Texture2D` for UI.

### 3.2 Generate PBR Maps Procedurally [MODIFY atlas.py]

For each block tile, generate 3 additional maps:

**Normal Map (128×128, RGB8):**
- Derive from the heightmap of the albedo texture. Use a Sobel filter:
  ```python
  # For each pixel (x, y):
  dX = luminance(albedo[x+1, y]) - luminance(albedo[x-1, y])
  dY = luminance(albedo[x, y+1]) - luminance(albedo[x, y-1])
  normal = normalize(vec3(-dX * strength, -dY * strength, 1.0))
  # Encode: (normal * 0.5 + 0.5) * 255
  ```
- `strength` parameter per block type: stone=1.5 (rough), glass=0.1 (smooth), wood=1.0 (medium).
- For bricks and stone_bricks: manually paint deeper grooves in the mortar lines for pronounced normal mapping.

**MRAO Map (128×128, RGB8 — packed):**
- **R = Metallic**: 0.0 for most blocks. 0.3–0.5 for ores (gold_ore=0.6, iron_ore=0.4). 0.0 for wood/dirt/stone.
- **G = Roughness**: 0.9 for dirt/cobblestone (very rough), 0.5 for stone/wood, 0.1 for glass (very smooth), 0.2 for water surface, 0.95 for leaves.
- **B = AO (micro-AO)**: Pre-baked ambient occlusion within the texture itself. Darken mortar lines in bricks, cracks in stone, wood grain channels. Complements the per-vertex AO from the mesher.

**Emissive Map (128×128, RGB8):**
- Most blocks: all black (no emission).
- Glowstone: bright yellow-white pattern matching the albedo cracks.
- Lava: orange-red flowing pattern (animated — see 3.4).
- Torch flame: small bright spot at the top.

### 3.3 Create Multi-Layer TextureArrays [MODIFY atlas.py, renderer.py]

Instead of one `TextureArray`, create separate arrays for each PBR channel:

```python
self.albedo_array = ctx.texture_array((128, 128, num_tiles), 4, data=albedo_data)
self.normal_array = ctx.texture_array((128, 128, num_tiles), 3, data=normal_data)
self.mrao_array   = ctx.texture_array((128, 128, num_tiles), 3, data=mrao_data)
self.emissive_array = ctx.texture_array((128, 128, num_tiles), 3, data=emissive_data)
```

Bind all four arrays to different texture units in the chunk shader:
```glsl
layout(binding = 0) uniform sampler2DArray u_albedo;
layout(binding = 1) uniform sampler2DArray u_normal;
layout(binding = 2) uniform sampler2DArray u_mrao;
layout(binding = 3) uniform sampler2DArray u_emissive;
```

Enable mipmaps + anisotropic filtering (8x or 16x) on all arrays to prevent texture shimmer at oblique angles.

### 3.4 Animated Textures [NEW section in atlas.py]

Some textures should animate:

- **Water surface**: 32 frames of scrolling/distorted noise. Store as 32 consecutive layers in the albedo array. In the shader, interpolate between `frame_n` and `frame_n+1` based on `u_time` (smooth crossfade).
- **Lava surface**: Similar to water but with slower, more viscous-looking animation (8 frames).
- **Torch flame**: 4-frame flicker cycle.

Pass `u_time` (already available) to the shader. Compute frame index: `int(u_time * fps) % num_frames`.

### 3.5 Connected Textures [NEW section in atlas.py + mesher.py]

Implement connected textures for glass and similar blocks:

- When two glass blocks are adjacent, the shared face should use a borderless texture variant.
- Generate additional atlas tiles: `glass_connected_full`, `glass_connected_edge_top`, `glass_connected_edge_left`, etc. (or use a single sheet with CTM variants).
- In `engine/graphics/mesher.py`, when emitting a face for glass, check the 4 edge-adjacent neighbors of the same block type. Select the correct connected texture variant based on which edges are shared.
- **Simpler approach (recommended):** Just generate a "glass_inner" tile without the border frame, and use it when the neighbor is also glass. This covers 90% of the visual benefit with minimal code.

### 3.6 Optional External Texture Pack Loading [NEW]

Add support for loading texture packs from a folder:

```python
# In atlas.py or a new textures.py:
class TexturePackLoader:
    def load_pack(self, pack_path: Path) -> dict[str, np.ndarray]:
        """
        Loads textures from pack_path/blocks/
        Expected structure:
          pack_path/
            blocks/
              stone.png           # albedo
              stone_n.png         # normal map
              stone_mrao.png      # metallic/roughness/ao
              stone_e.png         # emissive (optional)
            pack.json             # metadata: name, resolution, author
        """
```

- If a pack is found at `textures/` or configured in settings, load PNGs using Pillow.
- If a texture is missing from the pack, fall back to the procedural generator.
- Resize all loaded textures to a consistent resolution (use the pack's native resolution or downscale to 128×128).
- **Config integration:** Add `"texture_pack": null` to `configs/settings.json`. If set to a path, load that pack.

### 3.7 Block Definition Expansion [MODIFY configs/blocks.json]

Add PBR-relevant fields to each block definition:

```json
{
  "name": "stone",
  "textures": {"all": "stone"},
  "pbr": {
    "metallic": 0.0,
    "roughness": 0.75,
    "normal_strength": 1.2,
    "emissive_intensity": 0.0
  }
}
```

The procedural generator reads these values to create the MRAO and normal maps with correct parameters per block.
