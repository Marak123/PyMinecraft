## Implementation Steps

### 6.1 Structure Definition Format [NEW: `configs/structures/`]

Define structures as JSON schematics:

```json
// configs/structures/village_house_small.json
{
  "name": "village_house_small",
  "size": [7, 6, 7],
  "origin": [3, 0, 3],
  "biomes": ["plains", "desert", "snowy"],
  "surface": true,
  "blocks": [
    // Y-layer 0 (floor)
    [["cobblestone", "cobblestone", ...], ...],
    // Y-layer 1 (walls)
    [["oak_planks", "air", ...], ...],
    // ... etc
  ],
  "replacements": {
    "desert": {"oak_planks": "sandstone", "oak_log": "sandstone"},
    "snowy": {"oak_planks": "spruce_planks", "oak_log": "spruce_log"}
  },
  "loot_table": "village_chest"
}
```

### 6.2 Structure Generator [NEW: `engine/world/structures.py`]

```python
class StructureGenerator:
    """Places pre-defined structures in the world during generation."""
    
    def __init__(self, seed, registry):
        self.structures = self._load_all_structures()
    
    def place_structures(self, blocks, cx, cz, height_field, biome_field):
        """Called during chunk generation, after terrain fill, before trees."""
```

**Placement algorithm:**
1. **Candidate selection**: For each chunk, hash `(seed, cx, cz, structure_type)` to determine if a structure spawns. Use different frequencies per type:
   - Villages: 1 per ~400 chunks in plains/desert
   - Dungeons: 1 per ~100 chunks underground
   - Mineshafts: 1 per ~150 chunks underground
   - Ruined towers: keep existing logic (~0.11% per column)
2. **Jigsaw assembly** (for villages): Generate a village from multiple connected structure pieces:
   - Start with a village center (well, square).
   - Add roads in 4 cardinal directions.
   - Along roads, place houses, farms, blacksmiths at intervals.
   - Each piece snaps to a "jigsaw connector" (entry point with expected facing).
3. **Cross-chunk handling**: Structures larger than 16×16 write into a "pending structures" buffer. When the neighboring chunk generates, it checks for pending writes.
4. **Biome variants**: Use `replacements` dict to swap block types per biome.

### 6.3 Structure Types

**Village (surface):**
- Components: small house (7×6×7), large house (9×8×9), farm (7×4×9), blacksmith (9×7×9), well (5×5×5), church (9×12×7), road segments.
- NPCs: Villagers (Phase 7 — mob system). For now, just build the structures.
- Loot: Chests with random items (requires chest block — add as a non-interactable decoration block initially).

**Dungeon (underground):**
- Single room: 7×7×5 cobblestone/mossy cobblestone box.
- Contains: mob spawner block (decorative until hostile mobs in Phase 7), 1–2 chests.
- Generated inside stone at y=10–50. Must not overlap caves (check for existing air).

**Mineshaft (underground):**
- Network of 3-block-wide corridors at y=20–50.
- Features: oak plank supports (pillars + beams), rails (decorative block), spider webs (new cross-type block), ore deposits.
- Generated by random walk: start point → random direction → corridor length (8–20 blocks) → branch.

**Stronghold (underground, rare):**
- One per world (or per 1000-block radius). Contains the End portal frame.
- Multiple rooms connected by corridors. Libraries with bookshelves, prison cells.

### 6.4 Dimension System [NEW: `engine/world/dimensions.py`]

```python
class DimensionManager:
    """Manages multiple parallel worlds (dimensions)."""
    
    def __init__(self, seed, registry, storage_root):
        self.dimensions = {
            'overworld': Dimension('overworld', OverworldGenerator, ...),
            'nether': Dimension('nether', NetherGenerator, ...),
            'end': Dimension('end', EndGenerator, ...),
        }
        self.active = 'overworld'
    
    def switch_dimension(self, name, spawn_pos):
        """Switches the active dimension. Unloads current chunks, loads new."""
```

Each `Dimension` owns its own `World` instance, generator, and save directory. Switching dimensions:
1. Save current dimension's player position.
2. Unload all current chunks from GPU.
3. Set active world to the target dimension.
4. Begin streaming around the target spawn position.

### 6.5 Nether Dimension [NEW generator in generation.py or separate file]

**Characteristics:**
- Constrained height: y=0–128 with a bedrock ceiling at y=127 and floor at y=0.
- Terrain: Netherrack fills most of the space. Large open caverns carved by 3D noise.
- Features: Lava oceans at y=31. Glowstone clusters on ceilings. Soul sand valleys. Nether fortresses (large bridge-like structures).
- No sky light — only block light (torches, glowstone, lava glow).
- Red-tinted fog, no day/night cycle.

**New blocks needed:** netherrack, soul_sand, nether_bricks, nether_quartz_ore, crimson_nylium, warped_nylium, shroomlight, magma_block (emissive, damages on walk).

### 6.6 End Dimension [NEW generator]

**Characteristics:**
- Void below y=0 and above y=128. 
- Central island: large endstone platform at y=48–64 with obsidian pillars.
- Outer islands: floating endstone islands generated by noise with chorus plants.
- Dark purple sky. No day/night cycle. Only block light.
- Sparse ambient light (dim level 3 everywhere).

**New blocks needed:** endstone, obsidian, end_portal_frame, chorus_plant, chorus_flower, purpur_block.

### 6.7 Portal System [NEW section in game/game.py, NEW blocks]

**Nether Portal:**
- Built by the player: 4×5 obsidian frame (minimum), ignited by interaction (or a new "flint_and_steel" item from Phase 8).
- Portal blocks fill the inside of the frame (new transparent block with purple swirling texture, emissive).
- Walking into portal block triggers dimension switch after 4-second animation (screen tints purple, fades).
- Nether coordinate mapping: Overworld (x, z) → Nether (x/8, z/8).

**End Portal:**
- Found in strongholds. 12 end_portal_frame blocks in a ring. Must be "filled" with ender eyes (Phase 8 item).
- Portal activates when all 12 frames are filled. Teleports to End central island.
